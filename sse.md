# SSE에 대해 알아보자.

<br />
<br />

* SSE(Server-Sent Events)
---

```
SSE(Server-Sent Events)는 HTTP 프로토콜을 사용하여,
서버에서 클라이언트로 단방향으로 실시간 데이터를 푸시하는 기술이다.

웹 페이지가 새로고침 없이 서버로부터 실시간으로 업데이트된 데이터를 받을 수 있게 해준다.
```

1. 간단 요약

| 구분 | 설명 |
|---|---|
| **기술 명칭** | Server-Sent Events (SSE) |
| **통신 방식** | 서버에서 클라이언트로 단방향 통신 (Server-to-Client Push) |
| **기반 프로토콜** | HTTP |
| **연결 방식** | 클라이언트가 서버에 영구적인 HTTP 연결을 설정 |
| **데이터 형식** | 일반 텍스트 기반의 이벤트 스트림 형식 (Content-Type: text/event-stream) |
| **주요 이점** | 실시간 데이터 전송 용이, 웹소켓 대비 간단한 구현 및 오버헤드 감소 |
| **주요 사용 사례** | 실시간 알림, 뉴스 피드, 주식 시세, 로그 스트리밍 등 |
| **클라이언트 구현 (JavaScript)** | `EventSource` API 사용 |
| **서버 구현 (예시, Node.js)** | `response.writeHead`, `response.write` 등을 사용하여 `text/event-stream` 형식으로 응답 |
| **재연결** | 연결이 끊어질 경우 클라이언트가 자동으로 재연결 시도 |
| **장점** | 단순한 구조, 웹 표준 기반, 방화벽 친화적 |
| **단점** | 단방향 통신만 지원, 바이너리 데이터 전송에 비효율적 |
| **최신 동향 (2025년 기준)** | 웹소켓과 함께 실시간 웹 애플리케이션 구축에 활발히 사용, 브라우저 지원 확대 |
