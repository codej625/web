# sub domain이란?

<br /><br />

1. 서브 도메인(subdomain)
```
서브 도메인(subdomain)은 하위 도메인으로 웹사이트의 섹션을 구분하기 위해 도메인 이름에 추가되는 프리픽스(Prefix)를 말한다.
```

```
서브 도메인은 해당 메인 도메인과는 별도의 웹사이트로 작동한다. ex) cafe.naver.com, m.naver.com
이렇게 구분함으로써 웹사이트의 전반적인 목적성을 유지하면서 웹사이트 다양한 섹션을 개발할 수 있다. 
결과적으로 특정 분야에서 전문성을 구축하여 더 많은 유기적 트래픽을 얻을 가능성이 높아진다.
```

```
https://support.wix.com에서 "support" 부분이 서브 도메인이다.
```

<br />

2. 서브 디렉토리 (subdirectory) vs 서브 도메인
```
서브 디렉토리 (subdirectory)는 서브 폴더(subfolder)라고 이해하면 쉽다.
예시 -> example.com/blog

검색 엔진은 하위 도메인을 해당 메인 도메인과 별개로 인식하는 반면 서브 디렉토리는 도메인 내부에 중첩된 조직 폴더로 인식한다.
서브 도메인 대신 서브 디렉토리 아래에 콘텐츠를 분리해 놓으면 메인 도메인을 지원하고 있는 것으로 검색 엔진에 표시 된다. 
따라서, 웹사이트의 전체를 보완할 목적으로 더 많은 콘텐츠를 구성하려 한다면 서브 도메인보다 서브 디렉토리를 활용하는 것이 그 목적에 부합할 수 있다.
```

3. WWW(World Wide Web)
```
1990년에 W3은 인터넷에 연결된 컴퓨터를 이용해 사람들과 정보를 공유할수 있는 거미줄 같이 얽힌 공간을 뜻하는 용어이다.
HTTP프로토콜을 기반으로 HTML 작성된 하이퍼텍스트를 페이지를 웹 브라우저라는 특정한 프로그램으로 읽을 수 있게 하도록 구성된것으로 말한다. 
1994년 부터 사실상 인터넷과 동의어 취습할정도로 가장 널리 보급된 인터넷시스템이다.
인터넷 초창기에는 반드시 이용해야 했으나, 지금은 있는 곳도 없는 곳도 있다.

위의 정의 처럼 www(Wolrd Wide Web)은 서브 도메인(Sub Domain)의 역할을 한다기 보다 메인 서비스로 연결해준다 생각하시면 된다.
이런 하나의 www을 이용하고 기타 여러 서브도메인을 만들어 사용한다.
서브도메인(Sub Domain)은  아무 제약 없이 내 마음대로 이름을 정해서 설정할 수 있다.
```
