# 리디렉션(Redirection)

<br /><br />

1. Redirection?
```
클라이언트의 요청이 웹 서버에서 처리되고 나면,
서버가 클라이언트를 다른 URL로 보내는 프로세스를 의미한다.

리디렉션은 클라이언트의 경로를 변경하거나,
사용자를 다른 도메인으로 전송하거나,
페이지를 재구성할 때 유용하게 사용될 수 있다.

일반적으로 리디렉션은 웹 사이트의 구조를 변경하거나,
트래픽을 관리하거나, 오래된 URL을 새로운 URL로 업데이트하는 등의 목적으로 사용된다.

* 예시
http로 접속시 https으로 Redirection.
```

<br />

2. HTTP 상태 코드 
```
리디렉션은 HTTP 상태 코드를 사용하여 클라이언트에게 새로운 위치를 알려준다.
몇 가지 일반적인 HTTP 상태 코드 중에는 다음과 같은 것들이 있다.

- 301 Moved Permanently: 요청한 리소스가 영구적으로 새로운 위치로 이동되었음을 나타낸다.
- 302 Found (또는 302 Moved Temporarily): 요청한 리소스가 일시적으로 새로운 위치로 이동되었음을 나타낸다.
- 307 Temporary Redirect: 요청한 리소스가 일시적으로 새로운 위치로 이동되었음을 나타내며,
  POST 요청에 대해 원본 요청의 본문과 함께 리디렉트된다.
```

```
리디렉션은 사용자 경험과 SEO(검색 엔진 최적화)에 영향을 미치므로 신중하게 관리해야 한다.
올바른 사용 및 관리 없이는 잘못된 리디렉션은 웹 사이트의 성능에 부정적인 영향을 미칠 수 있다.
```
