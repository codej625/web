# 기본 포트에 대해 알아보자.

<br /><br />

1. 기본 포트 번호
```
80 (HTTP) -> 월드 와이드 웹을 통해 웹 페이지를 제공하는 데 사용된다.
보통 이 포트를 통해 HTTP 프로토콜을 사용하여 클라이언트의 요청을 서버로 전달한다.

443 (HTTPS) -> 보안 소켓 계층(SSL)을 통해 암호화된 통신을 제공하는 HTTP의 보안 버전인 HTTPS에 사용된다.
주로 웹 사이트에서 개인 정보나 기밀 정보를 전송할 때 사용된다.

20 (FTP 데이터 포트) -> FTP 프로토콜에서 데이터 전송을 위한 보조 포트이다.
FTP는 제어 연결에는 포트 21을 사용하고, 데이터 전송에는 포트 20을 사용한다.

21 (FTP) -> 파일 전송 프로토콜(FTP)을 사용하여 파일을 전송하기 위한 표준 포트이다.
FTP는 파일 서버와 클라이언트 간에 파일을 업로드 및 다운로드할 수 있도록 한다.

22 (SSH) -> 안전한 셸(Secure Shell)을 사용하여 원격 시스템에 로그인할 때 사용된다.
주로 리눅스 및 유닉스 시스템에서 원격으로 접속하기 위해 사용된다.
```

<br />

2. http, https 접속 예시
```
* HTTP

http://www.google.com -> http protocol.(O)
http://www.google.com:80 -> 80 포트를 명시했다.(O)
http://www.google.com:443 -> 443 포트를 명시했다.(O)

http://www.google.com:81 -> 81 포트를 명시했다. 목적이 다른 포트이므로 접속이 안 된다.(X)
```

<br />

```
* HTTPS

https://www.google.com -> https protocol.(O)
https://www.google.com:443 -> 443 포트를 명시했다.(O)

https://www.google.com:80 -> 80 포트를 명시했다.(X)
https://www.google.com:444 -> 444 포트를 명시했다. 목적이 다른 포트이므로 접속이 안 된다.(X)
```

<br />

```
포트 번호가 명시되어 있지 않다면, 프로토콜(http or https)을 확인하고 80이나 443 포트를 붙여 실험해보자. 
포트를 명시하지 않았을 때와 똑같이 잘 접속될 것이다.
```
